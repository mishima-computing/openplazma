# Observatory Compare MVP

M8 adds a small read-only compare page for two local OpenPlazma Runs.

The compare workflow is local-only and static. It reads existing `.openplazma/runs/...` records and writes generated HTML under `.openplazma/observatory/compare/`. It does not edit Runs, start a server, sync to a cloud service, require an account, or fetch external data.

## Purpose

The compare page helps a local user inspect how two Runs differ after running the Python and local Notebook workflow.

It compares:

- metrics
- artifacts
- source metadata
- target metadata
- capability metadata
- limitations

StudyFlow, Scenario, and StudyTask artifacts are compared automatically as normal artifacts when local Runs log them.

## Local Workflow

From the repository root:

```sh
rm -rf .openplazma
python notebooks/examples/local_runstore_example.py
python notebooks/examples/local_tracking_notebook.py
```

Then choose two Run IDs from `.openplazma/runs/` and export the Observatory with a compare page:

```sh
python scripts/export-observatory.py --run-store .openplazma --compare OPR-YYYYMMDD-000001 OPR-YYYYMMDD-000002
```

Open:

```text
.openplazma/observatory/compare/OPR-YYYYMMDD-000001__vs__OPR-YYYYMMDD-000002.html
```

## Generated Output

The exporter writes:

```text
.openplazma/
  observatory/
    index.html
    runs/
      OPR-YYYYMMDD-000001.html
      OPR-YYYYMMDD-000002.html
    compare/
      OPR-YYYYMMDD-000001__vs__OPR-YYYYMMDD-000002.html
    assets/
      observatory.css
```

This output is local generated output and must not be committed.

## Metric Comparison

Metrics are compared by metric name. The MVP uses the latest logged value for each name.

The compare page shows:

- metric name
- Run A latest value
- Run B latest value
- numeric delta when both values are numeric int or float values
- status: `same`, `different`, `only_in_a`, `only_in_b`, or `non_numeric`

Boolean values are not treated as numeric deltas.

## Artifact Comparison

Artifacts are compared by artifact name and artifact type.

The compare page shows:

- artifact key
- Run A path
- Run B path
- Run A SHA-256
- Run B SHA-256
- status: `same_hash`, `different_hash`, `only_in_a`, or `only_in_b`

Artifact links use the Observatory artifact link safety policy. Links are local relative links generated by the exporter, and canonical resolution must remain inside the intended `.openplazma/runs/<run_id>/artifacts/` directory.

## Capability Comparison

The compare page displays:

- `capabilities.readData`
- `capabilities.writeArtifacts`
- `capabilities.runSimulation`
- `capabilities.submitComputeJob`
- `capabilities.readFacilityTelemetry`
- `capabilities.controlFacility`

Current generated Runs should keep `runSimulation`, `submitComputeJob`, `readFacilityTelemetry`, and `controlFacility` false. If a RunRecord contains an unsafe true value, the compare page displays it visibly instead of silently normalizing it.

## Public Demo Boundary

The GitHub Pages public demo remains browser/static and `STATIC_FIXTURE`-only. It does not inspect a user's local `.openplazma/` RunStore and does not generate local compare pages.

## Limitations

- Compares exactly two Runs.
- No charts yet.
- Local only.
- Read-only.
- Static HTML only.
- No cloud sync.
- No hosted service.
- No public data ingestion.
- No AI assist.
- No real hardware instructions.
- Not a validated fusion simulator.
- Not a reactor design tool.
- Not a real hardware control system.
