# 0.1-alpha.0 Limitations

OpenPlazma `0.1-alpha.0` is an alpha release for safe public review of the first local learning workflow.

## Current Limitations

- `STATIC_FIXTURE` data only.
- No public data ingestion.
- No AI assist.
- No grading or scoring.
- No cloud sync.
- No hosted Observatory.
- No accounts.
- No local JupyterLab launcher.
- No JupyterHub.
- No real hardware instructions.
- Not a validated fusion simulator.
- Not a reactor design tool.
- Not a real hardware control system.

## Why These Limitations Exist

The first alpha focuses on a clear, inspectable path:

Lab -> Workbench -> StudyFlow -> RunStore -> Observatory -> Compare.

Keeping the release `STATIC_FIXTURE`-only makes the workflow reproducible for reviewers and keeps the public demo within safe educational scope.

## Future Milestones

Future milestones may explore public data readiness, external review, and first public data prototypes.

Those milestones are not part of `0.1-alpha.0`.

## Intentional Safety Boundaries

The following are intentional boundaries, not missing setup steps:

- No real hardware instructions.
- No real hardware control.
- No validated simulator claim.
- No reactor design claim.
- No hazardous experiment procedures.
