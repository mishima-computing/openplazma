# OpenPlazma 0.1-alpha.0 Release Notes Draft

Status: draft

## Summary

OpenPlazma `0.1-alpha.0` is the first public STATIC_FIXTURE-only learning demo. It includes a contract-first TypeScript monorepo, a Real Signal Room Lab, a Python notebook bridge, a JupyterLite Workbench, and a GitHub Pages static demo.

Public demo:

```text
https://mishima-computing.github.io/openplazma/
```

## Included

- `apps/lab`: React and Vite Lab for browsing the static sample shot and signal.
- `packages/core`: TypeScript data contracts.
- `packages/schema`: Zod validation for contracts.
- `packages/data-client`: Static fixture data source.
- `packages/signal-viewer`: Simple signal chart component.
- `python/openplazma`: Python SDK skeleton for ExperimentContext, StudyRecord, static signals, and plotting.
- Local RunStore MVP for Python and local Notebook workflows.
- Notebook Tracking Integration for logging ExperimentContext, SignalSeries, StudyRecord artifacts, and signal metrics into the local RunStore.
- Observatory UI MVP for read-only local static HTML inspection of RunStore Runs, metrics, artifacts, events, source, target, and capabilities.
- Observatory Compare MVP for read-only static comparison of two local Runs.
- Scenario / Study Task Layer with the first `STATIC_FIXTURE` Read the Signal task.
- First Guided Study Flow MVP connecting Read the Signal to Notebook, local RunStore, Observatory, and Compare.
- First Tutorial Pack MVP with a mission-style Read the Signal browser tour and local technical path.
- `apps/workbench-lite`: JupyterLite Workbench with a static notebook.
- GitHub Pages deployment for the combined Lab and Workbench static site.
- CI checks for TypeScript, Python, repository hygiene, JupyterLite, and Pages build validation.

## Scope and Limitations

- Uses `STATIC_FIXTURE` data only.
- Does not fetch external fusion data.
- Does not implement FAIR MAST ingestion.
- Does not sync RunStore records to a cloud service.
- Does not provide a hosted Observatory service.
- Does not include AI assist.
- Does not include grading or scoring.
- Does not implement toy physics.
- Does not launch local JupyterLab.
- Does not include JupyterHub.
- Is not a validated fusion simulator.
- Is not a reactor design tool.
- Is not a real hardware control system.
- Does not include real hardware instructions or hazardous experiment procedures.
