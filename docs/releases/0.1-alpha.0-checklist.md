# 0.1-alpha.0 Release Checklist

Use this checklist before creating the `v0.1-alpha.0` tag or GitHub Release.

## A. Repository State

- [ ] `main` is clean.
- [ ] All CI checks pass.
- [ ] Pages deploy succeeds.
- [ ] No generated files are tracked.
- [ ] No obvious secrets are present.
- [ ] No hidden or bidirectional Unicode controls are present.
- [ ] No external product names are introduced.

## B. Public Demo

- [ ] Public demo opens: `https://mishima-computing.github.io/openplazma/`.
- [ ] Lab loads.
- [ ] Workbench opens.
- [ ] Notebook file loads.
- [ ] Sample ExperimentContext JSON loads.
- [ ] Sample Signal JSON loads.
- [ ] Public demo says or documents `STATIC_FIXTURE`-only scope.

## C. Local Technical Workflow

Run:

```sh
python scripts/run-guided-study-flow.py --run-store .openplazma --clean
```

- [ ] RunStore output exists.
- [ ] Observatory export exists.
- [ ] Compare page exists.
- [ ] Generated HTML has no external network references.
- [ ] Cleanup removes `.openplazma`.

## D. Tutorial

- [ ] Tutorial index exists.
- [ ] Read the Signal Tutorial exists.
- [ ] Browser path works as documentation.
- [ ] Local path works as documentation.
- [ ] Troubleshooting exists.
- [ ] Glossary exists.

## E. Workshop Pack

- [ ] Teacher-kit README exists.
- [ ] 45-minute workshop README exists.
- [ ] Facilitator guide exists.
- [ ] Participant handout exists.
- [ ] Session plan exists.
- [ ] Setup checklist exists.
- [ ] Safety and scope notes exist.
- [ ] Feedback form exists.
- [ ] Local technical appendix exists.
- [ ] Slide outline exists.

## F. Safety And Scope

- [ ] No real hardware instructions.
- [ ] No validated simulator claim.
- [ ] No reactor design claim.
- [ ] No real hardware control claim.
- [ ] No public data ingestion claim.
- [ ] No AI assist claim.
- [ ] No grading or scoring claim.

## G. Release Execution Readiness

- [ ] Release notes are ready.
- [ ] Tag name is decided: `v0.1-alpha.0`.
- [ ] Release title is decided: `OpenPlazma 0.1-alpha.0`.
- [ ] Release body source is `docs/releases/0.1-alpha.0.md`.
- [ ] Do not create the release in this PR unless explicitly instructed.
