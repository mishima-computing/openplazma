# Read The Signal 45-Minute Session Plan

Duration: 45 minutes

Audience: beginner learners, teachers, science communicators, workshop facilitators, and developer reviewers.

## Prerequisites

- Browser-only path: a modern browser and access to the public OpenPlazma demo.
- Local technical path: a local clone of the repository and Python available from the project setup.

## Materials

- Public demo: `https://mishima-computing.github.io/openplazma/`
- Participant handout.
- Setup checklist.
- Safety and scope notes.
- Optional local technical appendix.

## Learning Goals

- Identify visible changes in a `STATIC_FIXTURE` signal.
- Write an observation separately from a hypothesis.
- Understand that a Run can store artifacts and metrics locally.
- See how Observatory and Observatory Compare inspect local RunStore output.

## Timeline

| Time | Activity |
| --- | --- |
| 0-5 min | Framing and safety/scope. State that this is `STATIC_FIXTURE` only. |
| 5-10 min | Open the Lab and inspect the signal. |
| 10-20 min | Write one observation and one hypothesis. |
| 20-30 min | Notebook or local workflow demonstration. Use facilitator demo if participants are browser-only. |
| 30-38 min | Observatory and Observatory Compare demonstration. |
| 38-45 min | Reflection and feedback. |

## Facilitator Notes

- Keep the focus on observing and recording signal-like data.
- Use the participant handout to separate observation from hypothesis.
- Use the local technical path only when the group has enough setup time.
- Keep public demo limitations visible throughout the session.

## Optional Extensions

- Repeat the guided StudyFlow twice and compare two local Runs.
- Ask participants to identify which artifact records were created.
- Ask participants which metric was most useful and why.

## What Not To Do

- Do not add grading or scoring.
- Do not describe the fixture as public data ingestion.
- Do not claim validated simulation behavior.
- Do not describe any real hardware workflow.
- Do not provide hazardous experiment procedures.
